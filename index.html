日志组件Logback之Policy

发表于 2022-05-23 更新于 2022-06-20 分类于 java ， 日志组件 ， logback

概述

在日志组件logback中，我们知道有Logger，Appender，Encoder，Layout，Filter等重要组件。日常使用中，我们很多时候都会使用RollingFileAppender来做日志文件滚动归档。那么日志文件是什么时候去执行滚动归档操作的,又是怎么样去归档的呢，这就不得不谈一谈logback中的RollingPolicy与TriggeringPolicy。TriggeringPolicy解决了when的问题，告诉系统什么时候去进行日志文件归档。RollingPolicy解决了how的问题，告诉系统日志文件应该怎么样去归档。

在logback中提供了很多触发策略，我们可根据提供的属性进行个性化配置。在进行日志文件滚动归档的时候，大体可以分为两种策略：文件大小和时间。

日志组件Logback之Filter

发表于 2022-05-09 更新于 2022-06-14 分类于 java ， 日志组件 ， logback

概述

在前面的文章中，我们已经介绍了logback的Logger、Appender与Layout组件，通过上面的组件，我们可以轻松熟练的使用logback。logback除了上面的几种组件，还给我们提供了Filter组件。本篇文章主要介绍Filter组件。

在logback-classic中提供了两种类型的filter：regular filter和turbo filter。每个filter提供了decide(ILoggingEvent event)方法来决定日志事件是否需要处理，decide方法会返回一个FilterReply枚举值，包含：DENY、NEUTRAL和ACCEPT。多个filter通过串联的方式可以组成一组过滤规则。

  • DENY：日志事件会被立即丢弃，不再执行剩余的filter的decide方法

  • NEUTRAL：日志事件会被传递到下一个filter中，并通过执行decide方法来确认日志的处理策略

  • ACCEPT：日志事件会被立即处理，不再执行剩余的filter的decide方法

Spring Cloud Gateway 动态Header

发表于 2022-04-11 更新于 2022-06-23 分类于 java ， spring ， spring cloud ， spring cloud gateway

概述

在前面的几篇系列文章中，我们讲解了Spring Cloud Gateway的大体使用以及scg处理一个请求的大致流程。本文主要讲解在scg中修改header参数，在一个http调用中，有RequestHeader与ResponseHeader两种header参数。

GatewayFilterFactory 处理类型 描述
AddRequestHeaderGatewayFilterFactory 请求 增加header，如果请求header中有对应的name，则value会进行追加
SetRequestHeaderGatewayFilterFactory 请求 设置header，与AddRequestHeaderGatewayFilterFactory不同，如果请求header中有对应的name，则value会被覆盖
MapRequestHeaderGatewayFilterFactory 请求 转换header，将fromHeader中的值增加到toHeader中去，如果不包含fromHeader则不进行操作
RemoveRequestHeaderGatewayFilterFactory 请求 删除header
AddResponseHeaderGatewayFilterFactory 响应 增加header，如果响应header中有对应的name，则value会进行追加
SetResponseHeaderGatewayFilterFactory 响应 设置header，与AddResponseHeaderGatewayFilterFactory不同，如果请求header中有对应的name，则value会被覆盖
DedupeResponseHeaderGatewayFilterFactory 响应 去除重复header
RewriteResponseHeaderGatewayFilterFactory 响应 修改header
RewriteLocationResponseHeaderGatewayFilterFactory 响应 修改响应header中的Location
RemoveResponseHeaderGatewayFilterFactory 响应 删除header

日志组件动态日志

发表于 2022-03-21 更新于 2022-04-01 分类于 java ， 日志组件

概述

日志对于一个系统来说十分的重要，在前面的几篇文章中，我们介绍了java语言的日志框架关系以及logback与log4j2两个优秀的框架。虽然日志十分重要，但这也是一把双刃剑，如果使用不当，大量的日志输出将导致我们的磁盘使用率以及IO迅速提升，从而可能导致系统崩溃。所以日志等级的配置在实际使用中十分的重要，在开发环境，我们可以打印debug等级日志方便我们排查问题，但是在生产上，我们很多时候都会配置info甚至更高的等级，这样可以节省部分磁盘或者IO。但是当我们生产上出现问题时，我们需要快速的切换日志等级来排查问题，这时候我们希望不是通过停止服务然后修改配置文件再重启服务这样的操作来实现，我们能在不重启服务的情况下来动态的修改日志等级吗？答案是肯定的。这篇文章我们主要介绍一下各个日志框架的使用，重点梳理一下在不重启应用的情况下如何来实现日志等级的动态改变。
