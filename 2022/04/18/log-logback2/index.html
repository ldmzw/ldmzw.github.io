日志组件Logback之Logger

发表于 2022-04-18 更新于 2022-05-13 分类于 java ， 日志组件 ， logback 阅读次数：

概述

在前面的文章中，我们已经介绍了logback的使用。在logbakc中，有三个十分重要的组件：Logger、Appender和Layout。通过这三种类型组件的配合使用，开发者们能够根据消息的类型以及等级来记录消息，以及控制消息应该怎么样来格式化和记录到哪里。本篇文章主要介绍Logger组件。

源码解读

在logback的世界中，我们打印日志其实都是一个一个的logger来实现的。logger的源码我们可以在ch.qos.logback.classic.Logger查看，其主要包含以下几个属性：

  • name：日志的名称
  • level：日志的等级，ch.qos.logback.classic.Level
  • effectiveLevelInt：日志实际生效的等级，在一个logger中，level有可能为null，但是effectiveLevelInt是不会为空的
  • parent：父日志
  • childrenList：子日志
  • aai：日志的Appender集合
  • additive：日志是否追加到父日志的Appender
  • loggerContext：日志上下文

配置文件转化

logger的转化主要来自于我们配置文件中的<logger>标签。

在上面的配置中，<logger>标签会被配置成logger实例。name、level、additivity分别对应实例中的name、level、additive,<appender-ref>对应实例中的aai。实例中的parent与childrenList会根据logger的名称自动去填充。loggerContext对应的则是整个<configuration>配置文件。

解析对应关系表

配置文件标签 logger实例
<configuration> loggerContext
<logger> logger实例
<logger#name> name
<logger#level> level
<logger#additivity> additive
<appender-ref> aai

结语

logger作为直接面向开发者的重要组件，它的实现可以说是非常的精简，让使用人员在使用的时候非常的简单与方便。
