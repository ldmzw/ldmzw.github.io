Spring Cloud Gateway 动态Header

发表于 2022-04-11 更新于 2022-06-23 分类于 java ， spring ， spring cloud ， spring cloud gateway 阅读次数：

概述

在前面的几篇系列文章中，我们讲解了Spring Cloud Gateway的大体使用以及scg处理一个请求的大致流程。本文主要讲解在scg中修改header参数，在一个http调用中，有RequestHeader与ResponseHeader两种header参数。

GatewayFilterFactory 处理类型 描述
AddRequestHeaderGatewayFilterFactory 请求 增加header，如果请求header中有对应的name，则value会进行追加
SetRequestHeaderGatewayFilterFactory 请求 设置header，与AddRequestHeaderGatewayFilterFactory不同，如果请求header中有对应的name，则value会被覆盖
MapRequestHeaderGatewayFilterFactory 请求 转换header，将fromHeader中的值增加到toHeader中去，如果不包含fromHeader则不进行操作
RemoveRequestHeaderGatewayFilterFactory 请求 删除header
AddResponseHeaderGatewayFilterFactory 响应 增加header，如果响应header中有对应的name，则value会进行追加
SetResponseHeaderGatewayFilterFactory 响应 设置header，与AddResponseHeaderGatewayFilterFactory不同，如果请求header中有对应的name，则value会被覆盖
DedupeResponseHeaderGatewayFilterFactory 响应 去除重复header
RewriteResponseHeaderGatewayFilterFactory 响应 修改header
RewriteLocationResponseHeaderGatewayFilterFactory 响应 修改响应header中的Location
RemoveResponseHeaderGatewayFilterFactory 响应 删除header

RewriteLocationResponseHeaderGatewayFilterFactory

从名字就能很好的看出，RewriteLocationResponseHeaderGatewayFilterFactory是用来重写响应头中的Location的值，以摆脱后端特定的详细信息。它接受四个参数：stripVersion、locationHeaderName、hostValue、protocols。

源码

从源码中可以看出，stripVersion有三种模式

  • NEVER_STRIP：不截取version信息，即使请求中不包含version信息
  • AS_IN_REQUEST：只有当请求信息中不包含version信息时才进行截取
  • ALWAYS_STRIP：始终截取version信息，即使请求中包含version信息

配置

在上面的配置中，由于hostValue与protocols没有指定，所以采用默认值。hostValue默认从exchange.getRequest().getHeaders().getFirst(HttpHeaders.HOST)ServerHttpRequest的header中获取，protocols默认为https?|ftps?。

示例，我们假设请求与响应信息如下

原始请求信息：http://xx.example.com/aa/bb
原始响应信息：Location：http://yy.example.com/v3/cc/dd

RewriteLocationResponseHeader=AS_IN_REQUEST, Location, , Location：http://xx.example.com/cc/dd 1.未指定protocols，默认为https?|ftps?，所有的http与ftp请求都将重写location
2.未指定hostValue，所以取ServerHttpRequest的header中获取Host，得到http://xx.example.com/v3/cc/dd

2.未指定hostValue，所以取ServerHttpRequest的header中获取Host，得到http://xx.example.com/v3/cc/dd

2.指定hostValue为zz.example.com:8080，得到http://zz.example.com:8080/v3/cc/dd

大家可以去尝试一下当请求为http://xx.example.com/v2/aa/bb时，上面配置的返回情况，可以发现最后返回的响应头中，除了配置为ALWAYS_STRIP，其它情况下都不会对响应头中的version信息进行截取。

注意：

version信息需要紧跟在host后面，否则无效。如：http://a.example.com:8080/api/v3/aa/bb，虽然在path中有`/v3`，但是`scg`并不会将他做为`version`信息。

结语

ReqeustHeader与ResponseHeader，它们的操作大致相同，从源码中也可以看出，两者的区别不大。唯一不同的是，因为ServerWebExchange中的ServerHttpRequest是不可修改的，所以我们要修改请求中的header时需要修改ServerHttpRequest后重新构建ServerWebExchange。
